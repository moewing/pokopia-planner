## 🎮 游戏核心机制

### 地图系统（关键约束）

- **全游戏共 5 张地图**
- **每张地图上限 25 只宝可梦**
- **超限后果**：多余宝可梦会**不显示**，需要用甜甜蜜吸引才能看到。未显示的宝可梦**无法参与该地图的资源循环**。
- **规划策略**：
  - 同一条资源循环链的宝可梦必须在同一张地图
  - 不同循环链建议分散到不同地图，避免某个链条因地图溢出而断掉

### 空间布局系统（重要纠偏！）

**✅ 正确理解**：
- **最小格子**：4×4 空间 = 1 个居住地，可住 **6 只宝可梦**
- **🔑 每个 4×4 格子内部是独立的环境**——两个挨在一起的格子可以一个是"潮湿"，一个是"明亮"
- **相邻共用边**：2 个 = 4×7，3 个 = 4×10
- **居住地判定范围**：以格子为中心的 **12×12**
- **重叠区物品共享**：3 连排时，中间 **6×12** 重叠区的物品被 **18 只宝可梦全部共享**（不管它们住在哪个环境的格子里）

**❌ 错误理解（之前有过，需纠正）**：
- ~~"3 连排的 18 只必须喜欢同一环境"~~ → 每个格子各自独立环境
- ~~"按环境聚类分配宝可梦"~~ → 不需要，环境约束仅在单格子内

### 三个容易混淆的概念

| 概念 | 作用 | 字段 |
|---|---|---|
| **栖息地**（Habitat） | 搭建蓝图**吸引野生宝可梦出现**（阶段1发现） | `habitats` |
| **喜欢的环境**（Env） | 宝可梦住的 4×4 格子是这个环境就舒适 | `env` |
| **喜欢的事物**（Likes） | 居住地范围（12×12）内放这些物品提升舒适度 | `likes` |

⚠️ 特别注意：kkplay3c 用"滋润"表示环境，PDF 用"潮湿"。两者是同一个环境（已统一为"潮湿"）。"滋润"在游戏中是**宝可梦给植物浇水的特长**，不是环境。

### 资源循环（3 条）

| 循环 | 链条 | 产出 |
|---|---|---|
| 木材循环 | 乱撒小圆木 + 分类 + 伐木 | 小圆木 → 木材 |
| 红砖循环 | 乱撒软塌塌黏土 + 分类 + 点火 | 黏土 → 红砖 |
| 铁条循环 | 乱撒不可燃垃圾 + 分类 + 回收利用 + 点火 | 垃圾 → 铁 → 铁条 |

**额外一次性加工**（非循环）：
- 废纸 → 纸：玩家在环境中捡到废纸后，交给"回收利用"特长的宝可梦处理

## 📋 数据结构

```typescript
interface Pokemon {
  id: number;
  name: string;                    // 简体中文（PDF版本）
  name_tw: string | null;
  icon_url: string;                // pokopiamap CDN
  icon_url_fallback: string;       // PokeAPI GitHub CDN
  is_playable: boolean;            // 是否可入住居住地
  specialties: string[];           // 中文特长：栽培/乱撒/伐木...
  specialties_en: string[];        // 英文：Grow/Litter/Chop...
  littered_items: string[];        // 乱撒物（仅"乱撒"特长时有）
  env: string | null;              // 6 种喜欢环境之一
  taste: string | string[] | null; // 5 种喜欢口味之一
  likes: string[];                 // 喜欢事物标签
  type: string[] | null;           // 属性（待 scraper 补）
  time_of_day: string[] | null;    // 待补
  weather: string[] | null;        // 待补
  habitats: string[];              // 推荐栖息地（待补）
  notes: string[];
}
```
